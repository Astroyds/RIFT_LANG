Contribution flow

Contributions follow a structured RFC process with clear ownership and review stages.

1. Proposal

Submit an RFC with motivation, syntax impact, and implementation notes.

2. Review

Technical steering group evaluates alignment with the language roadmap.

3. Merge + release

Accepted changes land with documentation updates and versioned releases.

Governance model

RIFT governance emphasizes transparency, predictable cadence, and long-term stability.

Technical Steering Group

Maintains the language specification and approves RFCs.

Module Owners

Each standard module has an accountable maintainer for quality and API clarity.

Release Council

Coordinates versioning, deprecation policy, and migration guidance.

Roadmap timeline

Code of conduct (excerpt)

RIFT maintains a respectful, professional community for all contributors.

We expect: constructive feedback, clear technical reasoning, and collaborative problem solving.

We do not allow: harassment, personal attacks, or discrimination.

Issues are escalated through the governance channel to ensure consistent enforcement.

License

RIFT is distributed under a permissive, enterprise-friendly license with clear attribution requirements.

Official license text is maintained in the main repository and is applied to the compiler, tooling, and documentation.

Commercial usage is permitted with attribution, and contributions inherit the project license.
